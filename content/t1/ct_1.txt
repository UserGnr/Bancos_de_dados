Banco de dados relacional e não relacional

Banco de Dados Relacional
- Edgar Frank Codd em 1970 (“Relational Model of Data for Large Shared Data Banks”)
- Baseado no Modelo Relacional
- Dados guardados em tabelas que podem ou não estar relacionadas
- Pontos fortes:
    - Oferecem aos usuários validação, verificação e integridade dos dados
    - Controle de concorrência, recuperação de falhas e segurança, controle de transações, otimização de consultas, dentre outros
- Pontos fracos
    - Dificuldade em conciliar o modelo relacional com a demanda por escalabilidade cada vez mais frequente
    - Dificuldade em organizar os dados em um sistema distribuído trabalhando com particionamento de dados
- Exemplos de bancos de dados relacinais:
    - Oracle database
    - SQL Server
    - PostgreSQL
    - MySQL
    - MariaDB
    - Access
    - SQLite
    - etc

Banco de Dados Não Relacional (NoSQL)
- 1998
- Alternativa ao Modelo Relacional
- NoSQL: abreviação de Not Only SQL (não apenas SQL)
- O grande diferencial é a performance
- Não utilizar um modelo estruturado em tabelas
- Consegue trabalhar com os dados de forma mais rápida
- Promovem alta disponibilidade e maior escalabilidade
- Algumas implementações oferecem:
    - Particionamento e a replicação de dados
    - Sistemas baseados em armazenamento de chave/valor
    - Sistemas orientados a documentos
    - Sistemas orientados a coluna
    - Sistemas baseados em grafos
    - etc
- Exemplos de bancos de dados não relacionais:
    - Mongo MariaDB
    - Apache Solr
    - Oracle Coherence
    - RAVENDB
    - HBASE
    - CouchDB relax
    - Casssandra
    - riak
    - membase
    - etc
